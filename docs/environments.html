# Americas AEM Cloud Environments: Three environments would be provided by Adobe which are shared across partners. ## Shared Environment list: * Production(prod): Java bundle containing all core functionality like OSGi services, listeners or schedulers, as well as component-related Java code such as servlets or request filters. * Stage: Java based integration tests * UAT: Java based integration tests * Sandbox: User Acceptance Testing (UAT) When work passes through the internal testing phase and is ready for approvals, the team deploys it to a UAT server. Staging(Stage) Staging is for pre-deployment. It’s like a dress rehearsal. A staging server’s set up is like production with all production configurations and the team uses it to perform smoke testing. This ensures the code and everything works in a production configuration and architecture. Hotfixes would be deployed directly on staging environment before actual prod deployment. Production(prod) Production servers are the final location for all finished and approved work. When you deploy code to a production server, this means everyone has approved it to go live. At this stage, the work is considered complete and approved for widespread use. Working code should only be deployed to a production server after it has been tested and approved for going live. ## Per partner Environment list: We will deploy sandbox on Azure and provide to each partner.
