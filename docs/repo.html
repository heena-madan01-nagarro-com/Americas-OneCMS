Repository Structure:

Each brand will have its own code reposiory for development. As we know Adobe supports single code repository deployment so we would be creating one single code repository which is replica of adobe cloud repository. Each brand's code would be synced to this single code repository.

After complete code quality checks, a github sync action would be triggered to sync Adobe Cloud Git from this repository.

Code Flow:

  • Each brand will work on its local environment. We will provide one QA environment on Azure and brand can test their code on provided QA.
  • When QA and code quality passes then code will be pushed to single Git repository by pull request.
  • After each brand's code merge to single git repository, a quality check would be triggered which is equivalent to Adobe Cloud quality checks.
  • If code of each brand's is passed then it will be synced to Adobe Cloud repository after an action(would be defined later). If code fails due to any reason, mail would be triggered with report and its responsibility of brand to fix the necessary issues.
